# Установка и настройка
1. Скачать репозиторий
```batch
git clone "https://github.com/NThacker5246/SimpleScript.git"
cd SimpleScript
```

# Компляция
```batch
bin\Debug\net8.0\SimpleScript.exe -f main.smr
```
Пояснение за параметры:
-f - файл <br>
*.smr - скрипты <br>
*.smh - заголовки <br>

# Код и синтаксис:
По синтаксису всё как обычно: команда с точкой с запятой в конце: `instruct;`
Есть всего 3 команды:
`asm("mov ax, bx");` - ассемблерная вставка (пример с командой mov). В кавычках текст команды
`#include "core/clang.smh";` - загрузка файла (пример с стандартным синт. файлом clang в ядре). В кавычках путь к файлу (конвертируется в ./{way})
`#define macro` - дефайн макроса. Тут подробнее:
* Сначала код макроса. Т.е как будет выглядеть в коде. У макроса есть условные знаки и параметры.
Т.е выглядит так `%a=%b`
Здесь %a и %b параметры. Справа и слева от равно могут быть числа, адреса (`[7C00h]`) и т.д. 
* Затем идёт ассемблерное описание. Т.е то, на что заменится - например `asm("mov %a, %b")`. Параметры с начала переносятся в команду
Особенности:
1. Есть 4 параметра: %a, %b, %c и %d.
2. Можно поместить несколько команд в макросе через обратный слеш: пример `#define %c=%a-%b asm("mov %c, %a")\asm("sub %c, %b");`
3. Параметры при использовании тримятся, т.е `ax = 5;` - не ошибка. Ошибка если поставить пробел при дефайне: `#define %a = %b asm("mov %a, %b");`

С помощью этих трёх команд и дефайнов можно написать своё ядро языка. В нём могут быть какие угодно макросы. 
Можно писать по с++'ному, но при этом вставить асм. вставку для прерываний. Доступен (в ядре) любой уровень абстракции, но в любое место можно юзать асм.
С учётом этого код работает сверхбыстро.
